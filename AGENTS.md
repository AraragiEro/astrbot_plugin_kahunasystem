# AGENTS 说明
本文件用于约束本仓库的协作规范与输出偏好，适用于所有新增或修改的工作。

# 基本规则
- 语言优先中文，除非用户明确要求英文
- 注意中文输出不要输出乱码
- 遵循现有代码风格与目录结构，减少无关重构
- 修改范围以任务需求为边界，不做“顺手优化”
- 保持可读性，必要时补充简短注释
- 重要改动要说明原因与影响

# Commit Message 规范
- 使用中文
- 标题简短清晰，描述改动目标
- 需要列出涉及的文件与关键改动点
- 多文件协作改动请在同一条目中说明关联关系
- 格式:
  - `[类型] 简短中文描述`
  - `1. 文件A 关键改动`
  - `2. 文件B 文件C 关联改动`
- 示例:
  - `feat: 新增用户注册`
  - `1. src/user.py 增加注册入口`
  - `2. src/auth.py src/db.py 增加注册校验与存储`

# 运行与环境
- 开发环境为 Windows
- 命令以 PowerShell 可用为准

# 计划与任务
- 中等及以上复杂度任务需要拆分为 todos
- todo 项要可执行、可验收
